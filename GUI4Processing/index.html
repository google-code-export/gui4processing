GUI for Processing (G4P)

I first came accross Processing in December 2008 and was immediately impressed with the ease of producing simple 2D and 3D animations. I particularly like the Toroid example but the inability to change the various parameters that controlled the drawing at runtime was disappointing. I decided what was needed was a set of 'simple to use' 2D GUI components.

I downloaded the library template, fired up Eclipse, created a code repository at GoogleCode and started programming not realising that the Processing site featured 2 such libraries already. I tried these libraries but they didn't do everything I wanted so I decided to continue with G4P.

I produced a working version and there the project would have stagnated as other matters took my attention. Then out of the blue I get an email from Jeff, he was looking for a GUI library to work with PeasyCam, another Processing library. The main problem appeared to be combining 2D/3D graphics and since I had my GUI working fine with Toroid I was confident it would work. I made the library available and after a few problems with Java versions (my fault) Jeff was able to try it out. Unfortunately either the GUI transformed with the camera or the position of the mouse and buttons were not aligned (PeasyCam works with the origin in the screen centre). After more delving into the Processing source code and more programming the problems were resolved and you can see the reults by clicking on the PeasyCam and other examples below.
